# CONTEXTO PARA EL CHAT ESPECIALIZADO DE ESTUDIO SIMO
## PLAN COMPLETO: ASISTENTE HÍBRIDO CON LENGUAJE NATURAL (TF-IDF + EMBEDDINGS)

## 1. Objetivo final
Un asistente que entienda **sinónimos, contexto y forma natural de preguntar** (como explicarle a un niño de 10 años).  
Sin código quemado (`if/else`), sin comparaciones estrictas de palabras exactas.

## 2. ¿Cómo funciona el enfoque híbrido?
**Primera etapa (rápida y liviana):**  
- Se usa **TF-IDF mejorado** (con normalización y sinónimos básicos).  
- Responde al instante (<10 ms).  
- Ideal para preguntas comunes y directas.

**Segunda etapa (semántica y flexible):**  
- Si TF-IDF no encuentra respuesta con alta confianza, se usa **MiniLM (embeddings)**.  
- Entiende sinónimos, contexto, preguntas mal redactadas.  
- Más lento (~300-500 ms), pero mucho más natural.

**Ventaja del híbrido:**  
Lo común se responde rápido (TF-IDF).  
Lo complejo o ambiguo se responde bien (embeddings).  
No dependes de reglas rígidas.

## 3. Requisitos técnicos (gratuitos y locales)
- **Transformers.js**: Librería para ejecutar MiniLM en el navegador.
- **Modelo MiniLM**: `Xenova/all-MiniLM-L6-v2` (70MB, se descarga la primera vez, luego queda en caché).
- **Navegador moderno**: Chrome, Edge, Firefox, Safari (con WebGPU o WebAssembly).
- **Sin internet después de la primera carga**: El modelo se guarda localmente en el navegador (cache y IndexedDB).

**Costo: CERO. Todo local. Sin servidores, sin APIs de pago.**

5. Código completo del buscador híbrido (js/buscadorHibrido.js)

import { pipeline, env } from '@xenova/transformers';

// Configurar para usar caché local
env.localModelPath = './modelos/';
env.allowRemoteModels = true;

let embedder = null;
let faqs = [];      // se cargará desde faqs.js
let embeddings = []; // almacena vectores precalculados

// 1. Normalizar texto (minúsculas, sin tildes, sin puntuación)
function normalizar(texto) {
    return texto
        .toLowerCase()
        .normalize("NFD").replace(/[\u0300-\u036f]/g, "")  // eliminar tildes
        .replace(/[¿?¡!;:.,()\[\]{}<>'"]/g, ' ')          // puntuación por espacio
        .replace(/\s+/g, ' ')                              // múltiples espacios a uno
        .trim();
}

// 2. Diccionario de sinónimos básico (para TF-IDF)
const sinonimos = {
    "url": ["enlace", "direccion web", "pagina web", "sitio web", "dominio", "link"],
    "inscribirme": ["registrarme", "inscribir", "registrar", "apuntarme", "postularme"],
    "inscripcion": ["registro", "apunte", "postulacion", "matricula"],
    "requisitos": ["requisito", "necesario", "documentos", "papeles", "que necesito"],
    "puntaje": ["puntuacion", "nota", "calificacion", "resultado", "puntos"],
    "aprobar": ["pasar", "ganar", "superar", "lograr", "conseguir"],
    "opec": ["vacante", "puesto", "cargo", "empleo", "plaza", "oferta"],
    "cnsc": ["comision nacional del servicio civil", "entidad de carrera", "estado"],
    "simo": ["sistema de apoyo", "plataforma", "aplicativo", "portal"]
};

// 3. Expansión de sinónimos en la pregunta
function expandirSinonimos(pregunta) {
    let expandida = pregunta;
    for (let [clave, lista] of Object.entries(sinonimos)) {
        for (let palabra of lista) {
            const regex = new RegExp(`\\b${palabra}\\b`, 'gi');
            if (regex.test(expandida)) {
                expandida = expandida.replace(regex, clave);
            }
        }
    }
    return expandida;
}

// 4. Cargar FAQs y precalcular embeddings (solo una vez al inicio)
export async function inicializarBuscador(objetoFAQ) {
    faqs = objetoFAQ;
    console.log(`Cargando ${faqs.length} preguntas...`);

    // Cargar modelo MiniLM (esto se descarga la primera vez, 70MB)
    embedder = await pipeline('feature-extraction', 'Xenova/all-MiniLM-L6-v2');
    console.log("Modelo MiniLM cargado");

    // Precalcular embedding de cada pregunta del FAQ
    for (let i = 0; i < faqs.length; i++) {
        const resultado = await embedder(faqs[i].pregunta, { pooling: 'mean', normalize: true });
        embeddings.push(Array.from(resultado.data));
        if (i % 20 === 0) console.log(`Procesadas ${i + 1}/${faqs.length}`);
    }
    console.log("Todos los embeddings precalculados");
}

// 5. Búsqueda por TF-IDF mejorado (rápida, con sinónimos)
function buscarPorTFIDF(preguntaUsuario) {
    const normalizada = normalizar(preguntaUsuario);
    const expandida = expandirSinonimos(normalizada);
    const palabrasClave = expandida.split(/\s+/);

    let mejorCoincidencia = null;
    let mejorPuntaje = 0;

    for (let i = 0; i < faqs.length; i++) {
        const preguntaNormalizada = normalizar(faqs[i].pregunta);
        let puntaje = 0;
        for (let palabra of palabrasClave) {
            if (palabra.length < 3) continue; // palabras muy cortas no aportan
            if (preguntaNormalizada.includes(palabra)) {
                puntaje++;
            }
        }
        // Normalizar por longitud de la pregunta del FAQ
        puntaje = puntaje / Math.sqrt(preguntaNormalizada.split(/\s+/).length);
        if (puntaje > mejorPuntaje) {
            mejorPuntaje = puntaje;
            mejorCoincidencia = faqs[i];
        }
    }

    if (mejorPuntaje > 0.4) {
        return { respuesta: mejorCoincidencia.respuesta, puntaje: mejorPuntaje, metodo: "TF-IDF" };
    }
    return null;
}

// 6. Búsqueda semántica con embeddings
async function buscarPorEmbeddings(preguntaUsuario) {
    if (!embedder) {
        console.warn("Modelo no listo aún");
        return null;
    }
    const resultado = await embedder(preguntaUsuario, { pooling: 'mean', normalize: true });
    const vectorQ = Array.from(resultado.data);

    let mejorSimilitud = -1;
    let mejorIndice = -1;

    for (let i = 0; i < faqs.length; i++) {
        let similitud = 0;
        for (let j = 0; j < vectorQ.length; j++) {
            similitud += vectorQ[j] * embeddings[i][j];
        }
        if (similitud > mejorSimilitud) {
            mejorSimilitud = similitud;
            mejorIndice = i;
        }
    }

    if (mejorSimilitud > 0.6) {
        return {
            respuesta: faqs[mejorIndice].respuesta,
            similitud: mejorSimilitud,
            metodo: "Embeddings",
            preguntaOriginal: faqs[mejorIndice].pregunta
        };
    }
    return null;
}

// 7. Función principal que integra el híbrido
export async function responder(preguntaUsuario) {
    // Primera etapa: TF-IDF rápido
    let respuesta = buscarPorTFIDF(preguntaUsuario);
    if (respuesta && respuesta.puntaje > 0.5) {
        return respuesta; // ya es confiable
    }

    // Segunda etapa: embeddings más precisos
    if (embedder) {
        const respuestaSemantica = await buscarPorEmbeddings(preguntaUsuario);
        if (respuestaSemantica) {
            return respuestaSemantica;
        }
    }

    // No se encontró nada
    return {
        respuesta: "No entendí tu pregunta. Por favor, intenta con otras palabras o revisa el glosario.",
        metodo: "ninguno",
        similitud: 0,
        puntaje: 0
    };
}

async function obtenerRespuesta(pregunta) {

}

8. Limitaciones y consideraciones importantes
Aspecto	Detalle
Primera carga	Descarga modelo MiniLM (70MB). Puede tardar 30-60 segundos en conexiones lentas.
Memoria RAM	Una vez cargado, MiniLM ocupa ~200-300MB. Puede ser pesado en celulares viejos.
Respuesta TF-IDF	<10 ms, instantánea.
Respuesta embeddings	~300-500 ms después de la primera respuesta (ya con modelo en RAM).
Navegadores compatibles	Chrome 92+, Edge 92+, Firefox 90+, Safari 15.4+. No funciona en navegadores antiguos.
Almacenamiento local	El modelo se guarda en caché e IndexedDB. No se descarga de nuevo si ya está.
9. ¿Qué ganas con esto?
Lenguaje natural real: "para qué sirve esto" entenderá que es similar a "qué es ESTUDIO SIMO".

Sin código quemado: no tienes que prever todas las variaciones de preguntas.

Escalable: si añades más preguntas al FAQ, el asistente las entiende sin modificar reglas.

Controlable: TF-IDF permite responder rápido lo común; embeddings lo complejo.

10. Conclusión
Este plan elimina las comparaciones estrictas y las intenciones rígidas. El asistente entiende por significado, no por coincidencia exacta. Es gratuito, local (no envía datos a ningún servidor) y técnicamente viable hoy mismo.

Si lo implementas, ESTUDIO SIMO tendrá un asistente conversacional de verdad, no un simple buscador de palabras clave.
